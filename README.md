# 劳务费税费换算工具

第一版轻量网页工具：无登录、无数据库、上传即算、导出即走。

## 1. 功能范围

- 支持上传 Excel 批量计算；
- 支持网页手动录入；
- 支持从 Excel 复制粘贴到网页表格；
- 提供 3 个下载模板：
  - `基础数据模板`：客户正式导入使用，只保留表头和一行示例；
  - `逻辑测试模板`：用于测试计算规则，覆盖免税、跨档、跨月、跨年、同名不同身份证等正常场景，并包含 `预期结果说明` Sheet；
  - `异常测试模板`：用于测试校验提示，故意包含缺字段、非法月份、非法金额等错误场景；
- 上传异常数据时会一次性汇总全部错误行，方便集中修正；
- 按 `年份 + 月份 + 身份证号码` 累计；
- 计算过程使用高精度 Decimal；
- 页面展示和 Excel 导出保留 2 位小数；
- 导出 Excel 包含四个 Sheet：
  - `劳务费税费换算台账`：严格贴近原表结构，计算列输出最终数值；
  - `清晰版台账`：按业务阅读习惯重新组织，计算列输出最终数值；
  - `公式版台账`：严格贴近原表结构，K-X 计算列保留 Excel 公式；
  - `测试核对报告`：自动输出基础核对结果；使用逻辑测试模板时，会逐项展示各测试场景是否通过。

## 2. 输入字段

基础数据至少需要以下字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| 年份 | 是 | 参与累计 |
| 月份 | 是 | 参与累计，1-12 |
| 事业部 | 否 | 原样输出 |
| 省区 | 否 | 原样输出 |
| 报销人 | 否 | 原样输出 |
| 会计 | 否 | 原样输出 |
| 姓名 | 是 | 原样输出 |
| 身份证号码 | 是 | 参与累计 |
| 税后劳务金额 | 是 | 核心计算金额，必须大于 0 |

## 3. 测试模板覆盖范围

`逻辑测试模板` 覆盖以下正常计算场景：

| 场景 | 验证点 |
|---|---|
| 累计税后 ≤ 800 | 税前 = 税后，个税 = 0 |
| 同人同月累计跨过 800 | 累计后进入个税区间，单次个税按累计拆分 |
| 累计税后 ≤ 3360 | 税前 = （税后 - 160）÷ 0.8 |
| 累计税后 3360 - 21000 | 税前 = 税后 ÷ 0.84 |
| 累计税后 21000 - 49500 | 税前 = （税后 - 2000）÷ 0.76 |
| 累计税后 ＞ 49500 | 税前 = （税后 - 7000）÷ 0.68 |
| 增值税 500 起征点附近 | ≤500 不计增值税，＞500 按 1% 计增值税 |
| 同一身份证跨月份 | 月份不同，累计重新开始 |
| 同一身份证跨年份同月份 | 年份不同，累计重新开始 |
| 同名不同身份证 | 不按姓名累计，只按身份证累计 |
| 不同人同月 | 不同身份证互不累计 |

`异常测试模板` 覆盖以下校验场景：

- 年份为空；
- 月份为空；
- 月份为 13；
- 姓名为空；
- 身份证号码为空；
- 税后劳务金额为空；
- 税后劳务金额为 0；
- 税后劳务金额为负数；
- 税后劳务金额为文本。

## 4. 计算规则

计算链条：

```text
单次税后金额
→ 本月累计税后金额
→ 反推累计税前金额
→ 计算累计个税
→ 拆出单次个税
→ 计算单次增值税
→ 计算单次附加税
→ 计算累计含税开票金额
→ 拆出单次应开票金额
→ 计算单次应付款金额
→ 核对
```

### 4.1 税后反推税前

| 累计税后金额 | 反推累计税前金额 |
|---:|---|
| ≤ 800 | 税前 = 税后 |
| ≤ 3360 | 税前 = （税后 - 160）÷ 0.8 |
| ≤ 21000 | 税前 = 税后 ÷ 0.84 |
| ≤ 49500 | 税前 = （税后 - 2000）÷ 0.76 |
| ＞ 49500 | 税前 = （税后 - 7000）÷ 0.68 |

### 4.2 个税

个税按累计金额计算，再拆回单次：

```text
单次个税 = 当前累计个税 - 之前已扣个税
```

累计个税规则：

| 累计税前金额 | 个税规则 |
|---:|---|
| ≤ 800 | 0 |
| 800 - 4000 | （累计税前金额 - 800）× 20% |
| ＞ 4000 | 累计税前金额 × 80% 后按 20% / 30% / 40% 三档计算 |

### 4.3 增值税与附加税

```text
本次对应税前金额 = 单次税后金额 ÷ 本月累计税后金额 × 本月累计税前金额
```

| 本次对应税前金额 | 增值税 |
|---:|---:|
| ≤ 500 | 0 |
| ＞ 500 | 本次对应税前金额 × 1% |

```text
附加税 = 增值税 × 12% × 50% = 增值税 × 6%
```

### 4.4 开票、付款、核对

```text
累计含税金额 = 累计税前金额 + 累计增值税 + 累计附加税
单次应开票金额 = 当前累计含税金额 - 之前已开票金额
单次应付款金额 = 单次应开票金额 - 单次个税
核对 = 税后劳务金额 + 增值税 + 附加税 - 单次应付款金额
```

正常核对值应为 `0.00`。

## 5. 后端启动

```bash
cd backend
python -m venv .venv
source .venv/bin/activate  # Windows 使用 .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

健康检查：

```text
http://127.0.0.1:8000/api/health
```

## 6. 前端启动

```bash
cd frontend
npm install
npm run dev
```

打开：

```text
http://127.0.0.1:5173
```

## 7. 后端接口

| 接口 | 方法 | 作用 |
|---|---|---|
| `/api/health` | GET | 健康检查 |
| `/api/template` | GET | 下载基础数据模板 |
| `/api/test-template/logic` | GET | 下载逻辑测试模板 |
| `/api/test-template/error` | GET | 下载异常测试模板 |
| `/api/calculate/upload` | POST | 上传 Excel 并计算 |
| `/api/calculate/manual` | POST | 手动数据计算 |
| `/api/export` | POST | 导出最终 Excel |

## 8. 测试

```bash
cd backend
PYTHONPATH=. pytest -q
```

当前测试覆盖：

- 同人同年同月累计；
- 跨年/跨月重置累计；
- 基础模板生成和读取；
- 逻辑测试模板生成、读取和预期结果 Sheet；
- 异常测试模板一次性汇总多行错误；
- 导出四个 Sheet；
- 前两个 Sheet 导出计算列为最终数值而非公式；
- 第三个 Sheet 保留 Excel 公式；
- 第四个 Sheet 输出测试核对报告。
