# 劳务费税费换算工具

第一版轻量网页工具：无登录、无数据库、上传即算、导出即走。

## 1. 当前口径

- 正式导入模板以最新后台导出表格为准，旧的“身份证号码导入模板”不再作为正式模板。
- 系统从 `会议日期` 自动拆分 `年份` 和 `月份`。
- 系统使用 `讲者ID` 作为唯一累计标识。
- 系统使用 `劳务费（实付金额）` 作为税后金额入口。
- 累计维度为 `年份 + 月份 + 讲者ID`。

## 2. 功能范围

- 支持上传后台导出的 Excel 批量计算；
- 支持网页手动录入；
- 支持从 Excel 复制粘贴到网页表格；
- 提供 3 个下载模板：
  - `后台导出模板`：客户正式导入使用，字段与后台导出表格一致；
  - `逻辑测试模板`：用于测试计算规则，覆盖免税、跨档、跨月、跨年、同名不同讲者ID等正常场景，并包含 `预期结果说明` Sheet；
  - `异常测试模板`：用于测试校验提示，故意包含缺字段、非法日期、非法金额等错误场景；
- 上传异常数据时会一次性汇总全部错误行，方便集中修正；
- 页面支持 `测试模式`：开启后，结果预览会额外展示累计维度、税后反推档位、个税档位、本次对应税前、增值税规则、跨档提示和核对状态；
- 计算过程使用高精度 Decimal；
- 页面展示和 Excel 导出保留 2 位小数；
- 导出 Excel 包含四个 Sheet：
  - `劳务费税费换算台账`：严格贴近原表结构，计算列输出最终数值；
  - `清晰版台账`：按业务阅读习惯重新组织，计算列输出最终数值；
  - `公式版台账`：严格贴近原表结构，K-X 计算列保留 Excel 公式；
  - `测试核对报告`：自动输出基础核对结果；使用逻辑测试模板时，会逐项展示各测试场景是否通过；
- GitHub Actions 会在 push / pull request 时自动运行后端 pytest，防止后续修改破坏计算逻辑。

## 3. 导入字段

正式导入 Excel 使用后台导出字段。系统至少需要以下字段：

| 后台字段 | 是否必填 | 系统用途 |
|---|---|---|
| 会议日期 | 是 | 自动拆分年份、月份，参与累计 |
| 事业部 | 否 | 原样输出 |
| 发起人 | 否 | 映射为报销人 |
| 讲者姓名 | 是 | 原样输出 |
| 讲者ID | 是 | 唯一累计标识 |
| 劳务费（实付金额） | 是 | 作为税后劳务金额，必须大于 0 |

完整后台模板字段包括：会议ID、会议类型、事业部、领域、产品、发起人、发起人员工ID、主题、会议日期、会议开始时间、会议结束时间、有效听众人数、在库听众数、会议是否有效、（讲者DEMS编码）、讲者姓名、讲者ID、客户级别、劳务角色、讲者医院、讲者科室、劳务费（实付金额）、个税、劳务费（合计）。

## 4. 测试模板覆盖范围

`逻辑测试模板` 覆盖以下正常计算场景：

| 场景 | 验证点 |
|---|---|
| 累计税后 ≤ 800 | 税前 = 税后，个税 = 0 |
| 同一讲者ID同月累计跨过 800 | 累计后进入个税区间，单次个税按累计拆分 |
| 累计税后 ≤ 3360 | 税前 = （税后 - 160）÷ 0.8 |
| 累计税后 3360 - 21000 | 税前 = 税后 ÷ 0.84 |
| 累计税后 21000 - 49500 | 税前 = （税后 - 2000）÷ 0.76 |
| 累计税后 ＞ 49500 | 税前 = （税后 - 7000）÷ 0.68 |
| 增值税 500 起征点附近 | ≤500 不计增值税，＞500 按 1% 计增值税 |
| 同一讲者ID跨月份 | 月份不同，累计重新开始 |
| 同一讲者ID跨年份同月份 | 年份不同，累计重新开始 |
| 同名不同讲者ID | 不按姓名累计，只按讲者ID累计 |
| 不同讲者同月 | 不同讲者ID互不累计 |

`异常测试模板` 覆盖以下校验场景：会议日期为空、会议日期月份非法、讲者姓名为空、讲者ID为空、劳务费（实付金额）为空、金额为 0、金额为负数、金额为文本。

## 5. 页面测试模式

页面顶部提供 `测试模式` 开关。开启后，结果预览表会额外显示累计维度、税后反推档位、个税档位、本次对应税前、增值税规则、跨档提示和核对状态。

## 6. 计算规则

计算链条：

```text
单次税后金额
→ 本月累计税后金额
→ 反推累计税前金额
→ 计算累计个税
→ 拆出单次个税
→ 计算单次增值税
→ 计算单次附加税
→ 计算累计含税开票金额
→ 拆出单次应开票金额
→ 计算单次应付款金额
→ 核对
```

### 6.1 税后反推税前

| 累计税后金额 | 反推累计税前金额 |
|---:|---|
| ≤ 800 | 税前 = 税后 |
| ≤ 3360 | 税前 = （税后 - 160）÷ 0.8 |
| ≤ 21000 | 税前 = 税后 ÷ 0.84 |
| ≤ 49500 | 税前 = （税后 - 2000）÷ 0.76 |
| ＞ 49500 | 税前 = （税后 - 7000）÷ 0.68 |

### 6.2 个税

```text
单次个税 = 当前累计个税 - 之前已扣个税
```

累计个税规则：累计税前≤800 为 0；800-4000 为（累计税前金额-800）×20%；超过 4000 时，累计税前金额×80% 后按 20% / 30% / 40% 三档计算。

### 6.3 增值税与附加税

```text
本次对应税前金额 = 单次税后金额 ÷ 本月累计税后金额 × 本月累计税前金额
```

本次对应税前金额≤500 时增值税为 0；大于 500 时按 1% 计算。附加税 = 增值税 × 12% × 50%。

### 6.4 开票、付款、核对

```text
累计含税金额 = 累计税前金额 + 累计增值税 + 累计附加税
单次应开票金额 = 当前累计含税金额 - 之前已开票金额
单次应付款金额 = 单次应开票金额 - 单次个税
核对 = 税后劳务金额 + 增值税 + 附加税 - 单次应付款金额
```

正常核对值应为 `0.00`。

## 7. 后端启动

```bash
cd backend
python -m venv .venv
source .venv/bin/activate  # Windows 使用 .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## 8. 前端启动

```bash
cd frontend
npm install
npm run dev
```

## 9. 测试

```bash
cd backend
PYTHONPATH=. pytest -q
```

cd frontend
npm install
